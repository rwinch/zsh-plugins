= cursor-rules

A zsh plugin that provides a command to link your Cursor rules into the current directory's git repository. This ensures that `$git_root/.cursor/rules` is a symlink to your shared rules.

== Overview

When you are inside a git repository, you can run the `cursor-rules` command. The plugin will:

* Find the root of the repository (using `git rev-parse --show-toplevel`).
* Ensure `$git_root/.cursor/rules` exists as a symlink to `~/code/rwinch/.cursor/rules`.

This is useful for explicitly sharing common AI coding rules across your local git repositories without automatically applying them everywhere.

== Installation

With antigen:

[source,bash]
----
antigen bundle rwinch/zsh-plugins/cursor-rules
----

Or source the plugin directory so that `cursor-rules.plugin.zsh` is loaded by your framework.

== Usage

Run the following command while inside a git repository:

[source,bash]
----
cursor-rules
----

This will create or update the `.cursor/rules` symlink in the repository's root directory pointing to `~/code/rwinch/.cursor/rules`.

== Requirements

* Zsh
* Git (for `git rev-parse --show-toplevel`)
* The directory `~/code/rwinch/.cursor/rules` should exist; the plugin will display an error if it does not.
